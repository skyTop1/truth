# 赛博祭祖-App打包与真机回归清单

版本：v0.1  
更新时间：2026-04-15

## 1. 文档目的

这份清单只服务当前仓库已经落地的 `App` 方向准备工作，不讨论云端接口、账号体系或服务端同步。

当前目标只有两件事：

1. 确认当前仓库已经可以生成 `app-plus` 调试资源
2. 把正式云打包前仍需补齐的真实配置、素材和真机验证项列清楚

---

## 2. 当前已确认状态

截至当前仓库状态，以下事实已经确认：

- `pnpm run build:app-plus` 已可成功执行

## 3. 当前 App 能力映射

基于当前代码，App 端重点能力集中在下面几块：

### 3.1 主链路页面

- `/pages/index/index`：首页祭台
- `/pages/workbench/index`：祖域管理页
- `/pages-sub/cyber/ritual-entry`：仪式接入页
- `/pages-sub/cyber/ritual-result`：结果页
- `/pages-sub/system/theme-preview`：主题预览页

### 3.2 App 端重点能力

- 结果页使用 `uni.createCanvasContext` 与 `uni.canvasToTempFilePath` 生成海报
- 结果页在非 H5 分支使用 `uni.saveImageToPhotosAlbum` 保存海报到系统相册
- 祖域页使用 `uni.setClipboardData` 导出本地备份串
- 祖域页使用 `uni.getClipboardData` 读取备份串做导入恢复
- 自定义底部导航通过 `uni.redirectTo` 在主包页面之间切换

### 3.3 App 端重点 UI 风险点

- 顶部导航依赖 `--status-bar-height`
- 底部导航依赖 `env(safe-area-inset-bottom)`
- 仪式页与祖域页都包含 `input` / `textarea`
- 页面导航包含主包与分包跳转，需要真机验证返回栈和分包加载体验

## 4. 最小命令检查

每轮 App 相关改动后，至少执行：

```bash
pnpm run lint
pnpm run type-check
pnpm run build:app-plus
```

说明：

- 这只代表工程静态校验和资源构建通过
- 不代表已验证 Android / iOS 真机行为
- 真机运行仍需用 HBuilderX 导入 `dist/build/app`

---

## 5. HBuilderX 导入与调试

建议固定按下面顺序操作：

1. 执行 `pnpm run build:app-plus`
2. 打开 HBuilderX
3. 选择导入已有项目
4. 导入目录选择 `dist/build/app`
5. 连接 Android 真机或模拟器运行
6. 如需 iOS 验证，再切到对应运行环境

建议先以 Android 真机为第一验证目标，因为当前仓库尚未接入任何 iOS 证书或签名文件。

## 7. App 真机回归清单

### 7.1 启动与首页

检查项：

- App 首次启动不白屏、不崩溃
- 首页祭台信息正常渲染
- 首页统计区能读取本地祖域仓
- 重新冷启动后，最近主题预设和本地记录仍能恢复

### 7.2 主包与分包跳转

检查项：

- 首页进入仪式页正常
- 结果页返回仪式页、首页、祖域页的路径正常
- 祖域页进入历史结果页正常
- 主题预览页返回路径正常
- 分包页面首开、回退、二次进入不出现白屏或路径错误

### 7.3 海报生成与相册保存

检查项：

- 结果页首次进入可自动尝试生成海报
- 重新生成海报可成功刷新状态
- 保存到相册成功时，有明确提示
- 系统拒绝相册权限时，有明确提示
- 保存失败但文件仍存在时，可回退到图片预览
- 本地海报缓存失效后，可重新生成恢复

### 7.4 本地备份导出与导入恢复

检查项：

- 祖域页复制备份串成功
- 祖域页从剪贴板读取备份串成功
- 剪贴板读取失败时，有明确错误提示
- 导入有效 JSON 后，首页、祖域页和结果页能读取恢复后的最新状态
- 导入无效 JSON 时，不应污染现有本地数据

### 7.5 安全区与自定义导航

检查项：

- 自定义顶部导航不被状态栏遮挡
- 底部导航不被手势条或底部安全区遮挡
- 刘海屏、全面屏下页面内容不被导航覆盖
- 页面滚动时固定导航不抖动、不出现明显错位

### 7.6 输入框与键盘顶起

检查项：

- 仪式页祖先称呼输入框可正常聚焦与输入
- 仪式页文案 `textarea` 不被键盘遮挡
- 祖域页导入恢复 `textarea` 不被键盘遮挡
- 键盘收起后页面布局能恢复

### 7.7 本地数据稳定性

检查项：

- 完成一次仪式后，杀进程重开 App，本地记录仍在
- 已生成海报的记录重开后可恢复海报状态
- 清空祖域仓后，首页和祖域页回到空态
- 主题切换后，杀进程重开仍保持最近预设

## 8. 当前已知未验证风险

当前仍未实际验证的 App 路径：

- Android 真机相册授权首次拒绝后再次授权流程
- Android 真机剪贴板权限或系统限制差异
- iOS 真机相册读写与证书配置联动
- iOS 真机 `safe-area` 与自定义导航表现
- 大图海报在低内存机型上的生成稳定性

---

## 9. 建议的执行顺序

建议固定按下面顺序推进：

1. 先补真实 Android 包名和签名信息
2. 用 HBuilderX 导入 `dist/build/app` 做 Android 真机回归
3. 完成相册、剪贴板、键盘和分包跳转验证
4. 补 App 图标、启动图和商店文案
5. 再补 iOS 证书与 iOS 真机验证

这样可以先把当前最接近可落地的一端跑通，避免同时被 Android 与 iOS 两条发布链阻塞。

- 真实签名与包名补齐
- Android 真机主链路跑通
- 相册、剪贴板和键盘行为稳定
