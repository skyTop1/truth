# 赛博祭祖核心页面低保真结构

版本：v0.1  
更新时间：2026-04-14

## 1. 文档目标

本文档把上一版热点概念，继续推进到页面结构层。

这份文档只解决三件事：

1. 首页第一屏怎么让用户 3 秒内上头
2. 仪式流程怎么压缩到 30 秒内完成
3. 分享结果页怎么做到适合截图传播

---

## 2. 核心页面流

完整主链路：

1. 打开小程序
2. 看到赛博祭台首页
3. 点击“开始接入祖域”
4. 进入仪式流程
5. 完成一次赛博祭祖
6. 生成结果页
7. 截图 / 分享 / 再来一次

目标要求：

- 全链路不超过 4 个主要页面
- 不要求登录才能先体验
- 首次体验必须无学习成本

---

## 3. 首页低保真

### 3.1 目标

首页不负责解释产品逻辑。

首页只负责两件事：

- 震住用户
- 让用户点进仪式

### 3.2 结构草图

```text
┌────────────────────────┐
│ 赛博祭祖        HOT    │
│ 祖域接入中 · 清明热度中 │
├────────────────────────┤
│ [协议标签] [量子祠堂]   │
│ [家谱星链] [云端香火]   │
│                        │
│      悬浮全息灵牌       │
│    紫蓝光圈 + 烟雾      │
│  当前接入祖先节点信息    │
│  今日香火值 / 信号值     │
│                        │
│ [开始接入祖域]          │
│ [查看信号色板]          │
├────────────────────────┤
│ 今日可点亮仪式          │
│ [上传电子香火] [量子元宝]│
│ [点亮光子烛]            │
├────────────────────────┤
│ 全球香火值 / 共祭人数    │
├────────────────────────┤
│ 最近三条祖域动态         │
└────────────────────────┘
```

### 3.3 首屏必须有的元素

- 大标题
- HOT 状态
- 世界观短句
- 悬浮灵牌
- 今日数据
- 主 CTA

### 3.4 首屏不能有的东西

- 长文说明
- 登录注册拦截
- 传统信息录入表单
- 正经产品说明文档式内容

---

## 4. 仪式流程页低保真

### 4.1 目标

仪式流程必须短、爽、酷。

不是做成复杂表单。

### 4.2 页面结构

```text
┌────────────────────────┐
│ 返回      接入祖域仪式   │
├────────────────────────┤
│ 当前祖先节点：A-108      │
│ 信号同步中...            │
├────────────────────────┤
│ 选择你的仪式动作         │
│                        │
│ [电子香火]              │
│ 描述 + 动画反馈         │
│                        │
│ [量子元宝]              │
│ 描述 + 动画反馈         │
│                        │
│ [光子烛台]              │
│ 描述 + 动画反馈         │
├────────────────────────┤
│ 留下一句星际悼词         │
│ [输入框]                │
├────────────────────────┤
│ [完成接入]              │
└────────────────────────┘
```

### 4.3 流程节奏

- 第一步：点一个动作
- 第二步：可选再点 1 到 2 个动作
- 第三步：输入一句话
- 第四步：完成并生成结果

### 4.4 必须保留的反馈

- 点击动作立即发光
- 出现能量波纹
- 当前累计香火值增长
- 按钮文案从“未点亮”变成“已上传”

### 4.5 建议动作

- 上传电子香火
- 投送量子元宝
- 点亮光子烛
- 激活祖先信号

### 4.6 文案风格

不是传统口吻。

建议用这种半正经半抽象表达：

- 电子香火已上传
- 祖域同步完成
- 量子供品投送成功
- 当前祖先节点波动稳定

---

## 5. 结果分享页低保真

### 5.1 目标

结果页不是“完成提示页”。

结果页是传播页。

### 5.2 页面结构

```text
┌────────────────────────┐
│ 今日赛博祭祖已完成       │
├────────────────────────┤
│   发光灵牌 / 海报主视觉  │
│                        │
│ 祖先节点：A-108         │
│ 今日香火值：108         │
│ 投送供品：3            │
│ 当前称号：祖域接入员    │
├────────────────────────┤
│ 你向祖域上传了一次心意   │
│ 祖先信号强度超过 87% 用户│
├────────────────────────┤
│ [生成分享图]            │
│ [再祭一次]              │
│ [去看祖域]              │
└────────────────────────┘
```

### 5.3 分享图必须有的内容

- 炫酷主视觉
- 今天做了什么动作
- 一个数字型结果
- 一个可炫耀的称号
- 一个容易转发的金句

### 5.4 分享文案建议

- 我刚完成了一次赛博祭祖
- 已向祖域上传 108 点香火值
- 当前祖先信号强度极高
- 清明限定祖域接入完成

---

## 6. 状态设计

### 6.1 首页状态

- 默认首访
- 当日热点活动中
- 已完成一次祭祀
- 今日无活动但仍可进入

### 6.2 仪式流程状态

- 未开始
- 进行中
- 部分动作已完成
- 提交完成
- 提交失败

### 6.3 结果页状态

- 默认完成
- 可分享
- 分享成功提示
- 再祭一次入口

---

## 7. 小程序交互优先级

### 7.1 必须优先保证

- 首屏打开速度
- 动作点击反馈
- 截图画面完整度
- 主要按钮易点

### 7.2 可以后置

- 复杂家谱编辑
- 长文本档案
- 多人实时协作
- 重后台管理

---

## 8. 首阶段实现建议

### 8.1 第一波直接做

1. 首页赛博祭台
2. 仪式动作弹窗或独立页
3. 结果页原型

### 8.2 第二波再补

1. 供品页
2. 祖域页
3. 家谱星图

### 8.3 设计判断标准

判断页面是否合格，不看功能是否完整，先看：

- 截图好不好看
- 第一眼记不记得住
- 点一下爽不爽
- 愿不愿意转发
