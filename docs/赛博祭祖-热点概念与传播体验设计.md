# 赛博祭祖 Hot App / 小程序产品概念与传播体验设计

版本：v0.1  
更新时间：2026-04-14

## 1. 项目定义

这不是一个长期主义的数字家族档案产品。

这是一款：

- 强概念
- 强视觉
- 强分享
- 强话题
- 强第一眼冲击力

的热点型产品。

一句话定义：

> 用最炸裂的科技感，把“祭祖”这件极传统的事，做成一眼就想点开、截图、转发的赛博仪式体验。

这个产品的价值不在“真正解决严肃祭祖需求”，而在：

- 制造讨论度
- 抢情绪注意力
- 提供反差感
- 形成短期传播记忆点

---

## 2. 产品目标

### 2.1 真正目标

核心目标不是让用户认真维护族谱十年。

核心目标是：

1. 用户 3 秒内理解这个产品很离谱但很想试
2. 用户 30 秒内完成一次“赛博祭祖”
3. 用户 60 秒内生成一张可分享的赛博祭祖结果图
4. 用户愿意把截图发到群里、朋友圈或短视频平台

### 2.2 成功标准

- 首页打开第一屏有记忆点
- 核心仪式流程足够短
- 每个动作都足够酷
- 结果页足够适合截图传播
- 文案足够抽象又洗脑

---

## 3. 用户画像

### 3.1 核心用户不是“真实祭祖人群”

核心用户其实是：

- 喜欢抽象互联网文化的年轻用户
- 会转发离谱小程序的人
- 会为了“这玩意太怪了”点进去体验的人
- 喜欢科技感、未来感、反差梗的人

### 3.2 用户心理

用户打开它，不是为了严肃祭奠。

而是为了：

- 看看能有多离谱
- 截个图发朋友
- 体验“未来人怎么祭祖”
- 获得短暂的情绪刺激和社交谈资

---

## 4. 产品策略

### 4.1 核心策略

把“祭祖”做成一个未来世界观下的数字仪式体验。

不是传统祠堂 UI + 加点光效。

而是：

- 全息灵牌
- 量子香火
- 数据焚香
- 祖先信号接入
- 赛博供品上传
- 家谱星图

### 4.2 正确做法

- 把仪式动作做得很酷
- 把分享结果做得很有梗
- 把世界观做得足够完整
- 把文案写得半认真半抽象

### 4.3 错误做法

- 做成普通纪念 App 换个皮
- 做成长篇家族档案管理系统
- 上来就堆复杂表单和资料录入
- 做得太像真正宗教/迷信服务

---

## 5. 平台策略

### 5.1 小程序是主战场

热点产品第一优先级必须是小程序。

原因：

- 即开即用
- 更适合社交裂变
- 更容易承接微信群传播
- 更适合节日节点爆发

小程序承担：

- 第一波传播
- 快速仪式体验
- 分享卡片生成
- 邀请朋友一起共祭

### 5.2 App 是加分项，不是首发刚需

如果目标是吃热点，App 不是第一阶段必要条件。

App 更适合：

- 做更重的 3D / 动效包装
- 承接后续更完整世界观
- 做活动延展和留存容器

第一阶段建议：

- `小程序先冲热度`
- `App 作为第二阶段壳层或延展版本`

## 6. 世界观设定

### 6.1 世界观关键词

- 量子祠堂
- 云端祖域
- 灵能网络
- 香火协议
- 家谱星链
- 祖先信号

### 6.2 视觉故事

未来世界里，物理祠堂已经不够用了。

人们把祭祀上传到云端，在一座“数字祖域”中完成仪式：

- 灵牌变成全息面板
- 香火变成数据流
- 供品变成 NFT 风格的赛博贡品
- 家谱变成宇宙关系图谱

用户不是“去拜”，而是“接入祖先网络”。

这个说法会比“线上祭祖”更有记忆点。

## 7. UI Direction Summary

### 7.1 视觉路线

这次就不要克制。

要的是：

- 未来感
- 奇观感
- 神秘感
- 高反差
- 强记忆点

### 7.2 视觉关键词

- 黑曜宇宙
- 霓虹电子香火
- 全息牌位
- 紫蓝能量雾
- 金色数据火花
- 祖先信号扫描线

### 7.3 气质要求

不是“夜店科技风”。

而是：

- 神秘
- 庄严但离谱
- 华丽但不廉价
- 像科幻片里的电子祭坛

### 7.4 画面第一眼要有的东西

- 中央悬浮灵牌
- 缓慢升腾的能量烟雾
- 底部发光操作按钮
- 周围环绕的数据符文 / 家谱轨道
- 强对比标题文案

## 8. Layout And Responsive Plan

### 8.1 一级信息架构

建议底部结构：

1. `祭台`
2. `供品`
3. `祖域`
4. `家谱`
5. `我的`

### 8.2 首页：祭台

第一屏必须强冲击。

结构建议：

- 顶部：赛博祭祖标题 + 今日祭祀指数
- 中央：悬浮灵牌 / 祖先核心面板
- 中段：快捷动作按钮
- 下段：今日可点亮的赛博仪式
- 底部：家族共祭动态 / 全球香火值

### 8.3 供品页

这里是最适合传播梗图的页面。

供品不能只是花和香。

建议加：

- 电子香
- 全息莲花
- 离子贡果
- 量子纸钱
- 机械仙鹤
- 赛博元宝
- 光锥烛台

规则：

- 每个供品都要有视觉反馈
- 每个供品都可以生成分享卡

### 8.4 祖域页

这里做成“数字纪念馆宇宙空间”。

内容结构：

- 当前接入的祖先节点
- 可切换的数字灵牌
- 已完成仪式记录
- 家族成员共祭痕迹

### 8.5 家谱页

不做正经表格家谱。

做成：

- 星图关系网
- 发光节点
- 连线脉冲
- 点击进入人物灵牌详情

### 8.6 我的页

偏轻量：

- 我的祭祀次数
- 我的赛博供品收藏
- 我创建的分享卡
- 我的祖域身份称号

## 9. Component Behavior Spec

### 9.1 `AppCyberAltar`

用途：

- 首页核心祭台组件

内容：

- 中央灵牌
- 烟雾 / 光圈 / 扫描线
- 当前祖先名称
- 仪式入口按钮

状态：

- 待接入
- 已接入
- 仪式中
- 完成态

### 9.2 `AppRitualAction`

用途：

- 单个赛博祭祀动作

动作建议：

- 上传电子香火
- 点亮光子烛
- 投送量子元宝
- 激活先祖信号
- 留下星际悼词

交互要求：

- 点下后必须有明显能量反馈
- 有粒子扩散、光环、按钮电流感
- 1 秒内完成反馈，不能拖

### 9.3 `AppOfferingCard`

用途：

- 展示供品卡

要求：

- 每张卡都要有单独图标和科技命名
- 支持已拥有 / 未解锁 / 限定款三种状态

### 9.4 `AppAncestorNode`

用途：

- 家谱星图人物节点

视觉：

- 节点外圈脉冲
- 点击时放大
- 当前查看节点高亮

### 9.5 `AppSharePoster`

用途：

- 结果海报生成组件

输出内容建议：

- 今日已完成赛博祭祖
- 点亮了多少香火能量
- 使用了什么赛博供品
- 获得什么祖域称号

这个组件是传播核心。

## 11. 传播与增长设计

### 11.1 核心传播点

传播不是靠功能说明。

传播靠结果页和话题文案。

建议做：

- 今日赛博祭祖已完成
- 已向祖域上传 108 点香火值
- 你的祖先信号强度超越 87% 用户
- 已点亮量子灵牌
- 获得称号：`香火黑客` / `祖域接入员` / `光明守灵人`

### 11.2 适合截图的内容

- 炫酷祭台完成页
- 发光灵牌
- 祖域称号卡
- 全息供品收藏页
- 家谱星图一角

### 11.3 可做的轻社交

- 邀请好友一起共祭
- 看看谁给祖域加的香火值更高
- 家族房间共同点灯

### 11.4 不要做的传播方式

- 强制分享解锁功能
- 分享换奖励的明显诱导
- 需要连拉 5 人才点亮供品

这样很容易掉价，也有平台风险。

## 12. 审核生存线

这个项目可以“赛博”，但不能把自己包装成明确的迷信服务。

### 12.1 对外描述建议

建议包装为：

- 数字纪念互动
- 未来感仪式体验
- 节日创意互动产品
- 家族记忆 / 赛博纪念空间

不要把文案直接写成：

- 招魂
- 通灵
- 法事
- 改运
- 超度服务

### 12.2 内容表达建议

可以有：

- 仪式感
- 纪念感
- 供品互动
- 光效祭台
- 家谱宇宙

不要有：

- 明确承诺现实效果
- 封建迷信导向文案
- 恐怖灵异惊悚内容
- 诱导分享换权益

### 12.3 最稳妥的说法

产品外壳上用“赛博纪念”“云端祭台”“数字祖域”。

在用户内部体验里再把氛围拉满。

## 13. MVP 范围

### 13.1 第一版必须有

- 炸裂首页祭台
- 3 到 5 个核心仪式动作
- 赛博供品页
- 结果分享页
- 祖域称号体系
- 家谱星图雏形

### 13.2 第一版不要做

- 复杂的真实家谱管理
- 长链路资料录入
- 大量后台配置
- 太多传统纪念档案逻辑

### 13.3 核心原则

- 所有东西都要为“体验一次也值”服务
- 不要为了留存把流程做重
- 不要为了功能完整性牺牲第一眼冲击

---

1. `祭台首页`

5. `家谱星图页`

## 15. 实现建议

### 15.1 先做什么

先做最能决定成败的三件事：

1. 首页祭台视觉
2. 仪式动作动画反馈
3. 分享结果页

### 15.2 现有项目怎么接

当前项目已经有：

- `AppButton`
- `AppCard`
- `AppDialog`
- `AppNavBar`
- `AppTabBar`
- `AppIcon`

下一步建议新增：

- `AppCyberAltar`
- `AppRitualAction`
- `AppOfferingCard`
- `AppSharePoster`
- `AppAncestorNode`

### 15.3 主题改造建议

当前项目原主题偏沉稳蓝灰。

如果要切到这套方案，建议：

1. 保留现有主题架构不动
2. 新增一套热点主题 token
3. 优先改首页、祭台、tabBar、按钮、弹窗
4. 让第一屏先炸起来，再补全其余页面

---

## 16. 最终建议

这个项目如果真要成立，关键不是“真实祭祖需求”，而是“传统题材 + 赛博视觉 + 社交传播”的巨大反差。

所以路线必须够明确：

- 不要半庄重半克制
- 不要试图同时讨好严肃纪念用户
- 不要做成资料管理工具

而要坚定一点：

> 这就是一个让人忍不住点进去看、看完想截图、截图后想发给朋友的赛博祭祖产品。

一句话收束：

> 产品不是为了被认真使用很久，而是为了在短时间内被大量看见、讨论、体验和传播。
