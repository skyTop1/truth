# 赛博祭祖-本地优先回归检查清单

版本：v0.1  
更新时间：2026-04-15

## 1. 文档目的

这份清单只服务当前仓库已经落地的能力验证，不讨论云端接口、账号体系或远程审核。

当前回归目标只有一件事：

> 确保 `Truth` 在 `H5 + 微信小程序` 场景下，首页、仪式页、结果页、祖域页和主题切换仍然维持“纯本地 + 本地缓存 + 可恢复”的主链路。

---

## 2. 回归范围

本轮默认覆盖：

- 首页祭台
- 仪式接入页
- 结果页
- 祖域页
- 主题预览页
- 本地祖域仓读写
- 本地海报生成与恢复
- 本地备份导出与导入恢复

本轮不覆盖：

- 远程接口联调
- 账号登录
- 云端同步
- 服务端审核
- 未实际运行的平台能力背书

---

## 3. 最小命令检查

每轮改动后，至少执行：

```bash
pnpm run lint
pnpm run type-check
```

说明：

- 这是工程静态校验，不替代端内手工回归
- 如果本轮只改文档，也建议保留至少一次静态检查，避免仓库处于已知损坏状态

---

## 4. 主链路回归清单

### 4.1 首次进入与空祖域状态

前置条件：

- 清空 `truth.cyber-ritual.archive`
- 清空 `truth-theme-preset`
- 如需兼容迁移验证，可单独保留 `truth-theme-mode`

检查项：

- 打开首页后，首页应显示首次进入或空祖域相关提示
- 首页统计区不应出现脏数据或旧记录残留
- 首页应明确说明“内容默认只保存在本机”
- 进入祖域页后，应出现空态提示，而不是报错或白屏
- 从首页进入仪式页的路由应正常

### 4.2 仪式接入与本地写入

检查项：

- 仪式页可编辑祖先称呼
- 仪式页可输入自定义文案
- 仪式页可点亮动作卡片，并实时更新香火值、信号值和称号
- 未填写祖先称呼或未选择动作时，应有明确 toast 提示
- 完成接入后，应正常跳转到结果页
- 结果页首次进入时，应自动写入一条本地祭祖记录

预期结果：

- `recentAncestorName` 应更新为本次输入值
- 祖域页应能看到新增记录
- 首页统计和最近称号应同步刷新

### 4.3 结果页与海报状态

检查项：

- 结果页首次进入时，应自动尝试生成海报
- 海报生成中、已就绪、失败、缓存失效、保存受限等状态文案应可见
- 点击“重新生成分享图”后，应能更新本地海报记录
- 从祖域页重新打开历史结果时，应优先恢复已有记录与海报缓存
- 本地海报缓存失效时，不应丢失仪式记录本身

平台差异：

- `H5`：重点检查图片预览、缓存失效后的重生成提示
- `微信小程序`：重点检查保存到相册成功、拒绝授权、失败回退三类反馈

### 4.4 祖域页管理能力

检查项：

- 祖域页应显示本地记录数、祖先节点数、海报数
- 最近记录列表和最近海报列表应能正常展示
- 点击历史记录可重新打开结果页
- 点击“重开最近战报”在有历史时应直接进入最近记录对应结果页
- 删除单条记录时，应同时删除关联海报
- 删除单张海报时，应保留仪式记录并重置该记录的海报状态
- 清空祖域仓后，首页与祖域页都应回到空态

### 4.5 本地备份导出与导入恢复

检查项：

- 祖域页可导出格式化 JSON 备份串
- 导出后 `lastBackupAt` 应更新
- 粘贴有效 JSON 后，可执行导入恢复
- 导入恢复会覆盖当前设备本地祖域仓，而不是追加脏数据
- 导入完成后，首页、祖域页和结果页应读取恢复后的最新状态
- 粘贴无效 JSON 或缺字段 JSON 时，应给出明确错误提示

### 4.6 主题切换与持久化

检查项：

- 主题预览页可切换四套主题：`light`、`dark`、`altar-gold`、`stele-cyan`
- 切换后，页面容器和基础组件颜色应立即变化
- 重进页面或重新启动应用后，应恢复最近一次主题预设
- 若本地只存在旧 key `truth-theme-mode`，启动时应自动迁移到 `truth-theme-preset`
- 无渐变主题不应因为缺少变量而出现样式报错或明显缺色

### 4.7 异常缓存与恢复策略

检查项：

- 本地祖域仓为空时，应进入 `fresh` 状态
- 本地祖域仓结构损坏时，应自动回退为空祖域仓，并给出恢复提示
- 旧版祖域仓导入后，应升级到当前 `version = 2`
- 旧英文 badge 应能被兼容转换成当前中文显示
- 海报临时文件失效时，应允许重新生成恢复

## 5. 建议的手工验证顺序

建议固定按下面顺序回归：

1. 清空本地缓存，验证首次进入
2. 完成一次仪式，验证结果页自动写记录
3. 生成海报，验证海报状态与预览或保存
4. 进入祖域页，验证记录、海报、删除和清空
5. 导出备份，再执行导入覆盖恢复
6. 切换四套主题并重启应用，验证主题持久化

这样可以最短路径覆盖：

- 本地写入
- 本地读取
- 本地恢复
- 本地删除
- 主题持久化

## 6. 平台说明

### 6.1 H5

重点关注：

- 主题 class 是否同步到 `html / body`
- 海报预览是否正常
- H5 下本地缓存清空后的首页和祖域空态是否正常

### 6.2 微信小程序

重点关注：

- 分包页面跳转是否正常
- 相册保存权限提示是否准确
- 页面重进后本地记录与主题预设是否仍可恢复

### 6.3 App

当前仓库保留 `build:app-plus` 脚本，但本清单不把 `App` 端视为已验证平台。

如果后续要补 `App` 端回归，必须单独记录：

- 权限弹窗行为
- 文件系统与图片路径差异
- 安全区与导航栏表现

---

## 8. 当前结论

`Truth` 当前最重要的回归，不是接口是否通，而是下面四件事是否仍然成立：

- 本地数据写得进去
- 本地缓存读得出来
- 本地海报失效后能恢复
- 本地主题切换重进后还能保持

只要这四件事稳定，这个项目当前的“本地优先赛博祭祖原型”定位就是成立的。
